# Operating Model

## Primary User

Drax v1.0.0 is for a founder who already has a product, a buyer hypothesis, and a real path to purchase or contact. The plugin qualifies those facts before constructing an organic growth system.

## Capability Loop

1. Intake captures founder, product, buyer, proof, voice, constraints, and current evidence.
2. Strategy defines falsifiable content and channel hypotheses.
3. Editorial creates source-backed briefs and a dependency-aware calendar.
4. Production creates reviewable article, SVG, video, audio, and metadata manifests.
5. Distribution queues approved assets through a controlled adapter.
6. Measurement records results and recommends continue, change, scale, or stop.

## Canonical State

Seven human-readable Markdown files hold the operating decisions. Facts belong in one canonical file and are linked elsewhere. Execution records and asset manifests may use JSON, but they cannot silently overwrite the human decision record.

## Decision Rights

The system may research, draft, render previews, compare results, and recommend changes autonomously. Public publishing, credential changes, paid spend, destructive actions, and strategic commitments require accountable approval.

## Internal Capability Reuse

`conclave-cc` is a source library, not a runtime dependency. A capability is promoted into Drax only when it is necessary for the current outcome, has a clear owner, passes security review, and does not duplicate an existing capability.
